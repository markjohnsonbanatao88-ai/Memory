# Pandora Privacy Review

## Purpose

Protect sensitive personal memory and reduce unnecessary retention of private information.

## Review areas

- Memory capture candidates.
- Stored memory summaries.
- Retrieval context packs.
- Logs and diagnostics.
- Admin screens.
- Exported reports.
- Pruning candidates.

## Principles

- Store the minimum useful memory.
- Prefer summaries over raw sensitive text.
- Redact secrets and unnecessary PII.
- Keep sensitive relationship, legal, financial, and health-like information review-gated.
- Do not expose private memory through public routes.
- Do not mix fictional sexual/story preference memory into real-life profile or decisions.

## Checklist

- Is this memory necessary?
- Is raw text needed, or can a redacted summary work?
- Does this include secrets, credentials, IDs, financial data, or highly sensitive personal claims?
- Is namespace correct?
- Is access user-scoped?
- Is the retention/review path clear?

## Completion standard

Return privacy risk level: low, medium, high, or blocked, with mitigation steps.
